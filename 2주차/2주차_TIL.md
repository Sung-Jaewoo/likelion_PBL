# 📘 Today I Learned

## 1. 오늘 배운 내용  
- `grid`와 `flex`를 사용해서 카드 형태 레이아웃을 구성
- 여러 개의 카드를 리스트 형태로 배치하고, 일관된 디자인을 적용.
- `@media query`를 이용한 모바일, 태블릿, 데스크탑 환경에서의 각각 다른 레이아웃 적용.  
- 사용자 입장에서 정보를 쉽게 확인할 수 있도록 시각적인 계층 구조(제목, 역할, 설명)의 구성.

## 2. 핵심 정리 (내 언어로)
- 교훈: 단순히 화면을 만드는 것이 아니라, 다양한 환경에서 잘 보이게 만드는 것이 웹 개발이다!!!  
- CSS의 `grid`-> 레이아웃을 구성할 때 매우 효율적인 도구. => 행과 열을 기준으로 구조를 잡을 수 있기 때문에, 여러 개의 카드 요소를 정렬하고 반응형으로 확장하는 데 직관적이고 유지보수도 쉬웠음.
- `@media query`->굉장히 대단한 기능. => 같은 HTML 구조를 유지하면서도 다양한 환경에 대응할 수 있다는 점이 효율적이었음.

## 4. 느낀 점
처음에는 그냥 카드 몇 개를 배치하는 단순한 작업이라고 생각했는데, 실제로 해보니까 생각보다 고려해야 할 요소가 많았다. 특히 같은 화면이라도 디바이스에 따라 다르게 보여야 한다는 점이 처음에는 잘 와닿지 않았는데, 직접 `@media query`를 적용해보면서 그 중요성을 체감할 수 있었다.  
또한 `grid`를 사용하면서 레이아웃을 구성하는 방식이 훨씬 체계적이라는 느낌을 받았다. 단순히 요소를 나열하는 것이 아니라, 전체 구조를 먼저 생각하고 배치해야 한다는 점에서 설계적인 사고가 상당히 필요하다는 것을 느꼈다. 이번 과제를 통해 단순 구현을 넘어서, “사용자가 어떤 환경에서 보게 될지까지 고려하는 것이 웹 개발이다”라는 점을 느끼게 되었고, 앞으로는 더 구조적이고 완성도 높은 UI를 만드는 방향으로 발전하고 싶다. "결론:사용자 입장에서 많이 생각해볼 수 있는 시간이었다."
